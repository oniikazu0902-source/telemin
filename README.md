# 空間MIDIコントローラー「tElemin」プロジェクト総括ドキュメント

## プロジェクト概要と目指す姿
Webカメラと画像認識AIを用いて、空中で手を動かすだけでソフトウェアシンセサイザー（Vital）を演奏できる、次世代型の仮想MIDIコントローラー開発プロジェクトです。
従来のテルミン（音程と音量の制御）の枠組みを大きく拡張し、「絶対に不協和音にならない魔法の楽器」という思想をベースに設計されています。単に音を鳴らすだけでなく、「和音制限」「手の開閉による音量フェード」「左右の動きによる波形のリアルタイム変形（3Dコントロール）」「HUDによる視覚的フィードバック」を統合し、実用的なデジタル楽器としての完成度を追求しています。

## システムアーキテクチャ・技術スタック
システムの全体像と各コンポーネントの役割は以下の通りです。

| レイヤー | 採用技術・ライブラリ | 役割・備考 |
|---|---|---|
| 開発言語 | Python | 柔軟なロジック構築とAI連携の基盤 |
| 骨格検出AI | MediaPipe (Tasks API / HandLandmarker) | 手の21箇所の関節ポイント（ランドマーク）を毎秒高速に検出。動かすには `hand_landmarker.task` ファイルが必須。 |
| 画像処理・HUD | OpenCV (cv2) | カメラ映像 of 取得、鏡像反転、有効エリアやデバッグ情報のオーバーレイ描画 |
| MIDI通信 | mido / python-rtmidi | 計算した座標データをMIDI信号（Note On/Off, CC）に変換して送信 |
| 仮想MIDIケーブル | Windows MIDI Services (Default Basic App Loopback) | loopMIDIのWindows Updateによる不具合を回避するために導入した、Microsoftの次世代MIDIループバック規格。 |
| 音源（シンセ） | Vital（スタンドアロン起動） | OMNIモード（全チャンネル受信）を設定し、Pythonからの多次元入力を生演奏の音へと変換。 |

## コア機能と操作思想（3Dコントロール）
手の動きを完全にX、Y、Z（ピンチ）の3次元パラメータとして解釈し、シンセサイザーへ送信する多次元設計思想を採用しています。

### ── 【Y軸】音程制御と和音（スケール）制限
- **検出基準**: ユーザーの演奏スタイルに合わせてカスタマイズされた「親指の先端（ランドマーク4番）」のY座標。
- **操作思想**: 縦の座標をそのまま半音階（クロマチック）にするのではなく、あらかじめ定義した特定のスケール配列（`ALLOWED_NOTES`）にマッピングして発音。これにより、どこを触っても美しい旋律が響きます。
- **リアルタイム切り替え**: 演奏中にキーボードの `c`（ハ長調）、`a`（イ短調）、`g`（ト長調）を押すだけで、`KEY_PRESETS` の辞書から参照する配列を動的にすり替える機能を実装。
- **デッドゾーン設定**: カメラ端の誤認識による音の飛びやチャタリングを防ぐため、上下に有効エリア制限（`Y_MIN_LIMIT`, `Y_MAX_LIMIT`）を設置。

### ── 【Z軸・ピンチアクション】発音トリガー（Note On/Off）と音量（CC#7）
- **検出基準**: 親指（4番）と人差し指（8番）の2点間距離を `math.hypot` で計算。
- **操作思想**: 「狙った高さで指をつまむと音が鳴り、離すと消える」という、キーボードライクかつ直感的なトリガー操作。
- **反転・直感化ロジック**: 単なる距離測定では「指を離すほど数値が大きくなる」ため、`1.0 - (距離 × 6.0)` という計算式で数値を反転。指をくっつけた時に 100%、離すと 0% になるよう最適化。
- **トリガー制御**: 割り出した％がしきい値（`PINCH_THRESHOLD = 0.7`）を超えた瞬間に Note On を送信し、下回ると Note Off を送信して確実に消音。同時に、ピンチの強度をそのままメインボリューム（CC#7）として送信し、滑らかな音量のフェードイン・フェードアウトを可能にしています。

### ── 【X軸】音色・波形の動的変形（Wavetable Position）
- **検出基準**: 手の左右の動き（X座標）を抽出し、MIDIコントロールチェンジ（CC#16）に変換。
- **操作思想**: 「音色を空間で削り出す」。Vitalの「MATRIX」画面経由で CC#16 を「Macro 1」へ直接ルーティングし、Wavetable Position（波形の形状変化）をダイナミックにコントロール。手を左から右へ動かすことで、柔らかいサイン波からエッジの効いたノコギリ波へと音色をグラデーション変化させます。
- **モード切り替え**: キーボードの `w` キーを押すことで、この波形変化モードのON/OFFをいつでも演奏中に切り替え可能です。

## 実装における重要な工夫と技術的ブレイクスルー

### ① 両手独立のMIDIルーティング
MediaPipeで最大2つの手を同時に検出し、右手は「MIDIチャンネル1（channel=0）」、左手は「MIDIチャンネル2（channel=1）」と、送信レーンを完全に分離。これにより、右手のメロディ演奏と左手の伴奏（1オクターブ下げる `note - 12` のベース化ロジック）が干渉しない設計を実現しました。

### ② チャタリング（連打）防止とロングトーンの実現
ピンチ操作による発音時、「つまんだ瞬間の最初の音程（Note）」を保持（`if state['note'] is None:`）するロジックを実装。これにより、手を上下に動かしながらつまんでも「タラララ」と連打されず、ギターのチョーキングやシンセのポルタメントのように「じゃーーん」と綺麗に音が伸びる仕様を確立しました。

### ③ リアルタイムHUD（ヘッドアップディスプレイ）
OpenCVによる描画を活用し、有効範囲の境界線（赤ライン）、現在のキー設定、手の状態（ピンチ強度％）、現在発音中のMIDIノート番号を画面上にオーバーレイ表示。つまんだ瞬間に指先の円が「赤から緑」に変わる当たり判定デバッグ機能を搭載し、直感的な操作感の調整を可能にしました。

### ④ ワンクリック自動起動環境
複雑な起動プロセスを排除するため、Windowsバッチファイル（`.bat`）を作成。バッチファイルをダブルクリックするだけで、「Vital（シンセサイザー）の自動起動 ➔ 2秒待機（ポート独占防止） ➔ 仮想環境（`.venv`）の有効化 ➔ Pythonスクリプトの実行」が一発で完了する極上の演奏環境を構築しました。

### ⑤ 無人時のYMO「ライディーン」自動生演奏機能
カメラに手が映っていない（誰も演奏していない）状態を検知すると、`threading`（マルチスレッド）によって裏方スタッフを動かし、バックグラウンドで `rydeen.mid` を自動ループ演奏させる機能を実装。人が来ると演奏がスッと一時停止（pause）し、テルミンモードへシームレスに切り替わります。

## 開発における教訓とノウハウ

- **MediaPipe of 仕様変更トラップ**
  古い `mp.solutions.hands` の書き方は最新バージョンで廃止されたため、AIモデルファイル（`.task`）を別途読み込む最新の「Tasks API」アーキテクチャへとコードを刷新。ファイル読み込みエラーに対しては、プログラムと同じフォルダに置くか、完全な絶対パス（`r'C:\...'`）で指定するのが確実であるというノウハウを得ました。

- **MIDIポートの排他制御（VitalとPythonの起動順序）**
  WindowsのMIDIシステムは「1つのポートは同時に1つのソフトしか使えない」という排他制御仕様です。Python側からポートが消えるエラーを防ぐため、「ポート確認時は必ずVitalを完全に閉じる」「バッチファイルではVitalを先に起動し、後からPythonを起動する」という鉄則を解明しました。

- **CC信号のトラフィックジャム（渋滞）回避**
  Vitalの「Learn MIDI Assignment」機能を使う際、CC#7（ピンチ）とCC#16（X軸）が同時に高速送信されていると誤学習してしまう問題が発生しました。これを、Vitalの「MATRIX」画面（裏設定）で「SOURCEをCC#16、DESTINATIONをMacro 1」と直接手動で指定し、変化量（AMOUNT）を最大に振り切らせることで完全に解決しました。

- **OpenCVの「cv2.waitKey」による入力ブロック**
  キーボードの切り替え入力が1回しか効かなくなる問題に対し、`if key != -1:`（本当にキーが押された瞬間だけ判定する）という条件を挟み、`& 0xFF` でマスク処理を行うことで、何度でも確実にスケールを切り替えられるようにロジックを修正しました。

## 新環境への移行・拡張ガイド（今後のアップデート方針）
新環境への移行後、さらにこの楽器を進化させるためのロードマップです。

### 拡張性バツグンの「全部乗せ」アーキテクチャを維持せよ
現在のコードは、AIが手を解析する部分（`extract_hand_features`）と、音を鳴らすメインループが明確に分離（モジュール化）されています。今後はこの関数内に計算式を追加していくだけで、コード全体を壊すことなく安全に機能を追加できます。

#### 💡 次なる進化のためのMIDIコントロール候補
現在はX軸に Wavetable Position（CC#16） を割り当てていますが、手の傾きや別の指の動きを検出して以下の汎用CCにマッピングすることで、さらなる多次元演奏が可能になります。

- **Filter Cutoff（CC#74 / Brightness）**
  音の「こもり具合 / 明るさ」を調節します。手を動かすにつれて「モコモコした音」から「バキッとした明るい音」に変化する、DJライクなパフォーマンスに最適です。
- **Modulation Wheel（CC#1）**
  音を揺らす「ビブラート」の強さをコントロールします。音が「じゃーーん」と伸びているロングトーンの時に、手を小刻みに揺らすことで感情豊かな演奏が可能になります。
- **Resonance（CC#71）**
  フィルターの境目を強調し、音に独特のクセや「ミョンミョン」というシンセサイザー特うのうねりを加えたい時に有効です。
